Sobre esta Aplicação

Esta página descreve os detalhes técnicos, arquitetura e o processo de desenvolvimento deste portefólio.

1. Arquitetura MVT do Django

O Django utiliza o padrão de arquitetura MVT (Model-View-Template), que é uma variação do MVC (Model-View-Controller).

Desenho da Arquitetura (Papel e Lápis):

Nota: Carregue a fotografia do seu desenho para a pasta static

  • Model: Gere os dados e a lógica da base de dados. Define a estrutura das tabelas.
  • View: Contém a lógica de negócio. Recebe pedidos HTTP e decide que dados enviar para o Template.
  • Template: Define a camada de apresentação (HTML/CSS). É onde os dados são renderizados para o utilizador.

Fluxo: Pedido HTTP → URL Dispatcher → View → Model (opcional) → View → Template → Resposta HTTP.

2. Modelação

A estrutura de dados da aplicação baseia-se num Diagrama Entidade-Relação que define como as UCs, Projetos, Docentes e Tecnologias se interligam.

Diagrama Entidade-Relação (DER)

Implementação em models.py

class Projeto(models.Model):
    titulo = models.CharField(max_length=200, unique=True)
    unidade_curricular = models.ForeignKey(UnidadeCurricular, on_delete=models.SET_NULL, null=True, blank=True)
    descricao = models.TextField()
    tecnologias = models.ManyToManyField(Tecnologia)
    # ... outros campos
